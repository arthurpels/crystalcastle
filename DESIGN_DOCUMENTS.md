# Механика внутриигровых документов

**Движок:** Unity 2022.3 URP  
**Опирается на:** существующий `IInteractable` / `InteractionManager` / `PlayerInventory`

---

## Концепция

Документы — отдельный тип объекта, не предмет инвентаря. Они не занимают слот, не экипируются. Нашёл → прочитал → сохранилось в архив. Можно перечитать в любой момент из меню паузы.

Три типа документов по подаче:
- **Бумага** — записки, журналы, приказы. Полноэкранный оверлей с текстом.
- **Аудиокассета** — диктофонные записи. Воспроизводятся голосом поверх игры, игрок продолжает двигаться.
- **Составной документ** — дневник Горина, журнал Вороновой. Несколько частей, найденных отдельно. В архиве собираются в одну запись.

---

## Архитектура

### DocumentData (ScriptableObject)

```
DocumentData
├── id                  string        уникальный ID ("gorin_diary_part1")
├── title               string        название в архиве ("Дневник Горина, часть 1")
├── body                string        полный текст (поддерживает \n)
├── type                DocumentType  Paper / Audio / MultiPart
├── groupId             string        для составных ("gorin_diary") — у частей одинаковый
├── partIndex           int           номер части (0, 1, 2...)
├── totalParts          int           сколько всего частей
├── audioClip           AudioClip     только для Audio-типа
├── interactPrompt      string        текст подсказки ("Читать", "Поднять кассету")
└── ambientOnRead       AudioClip     необязательно — фоновый звук во время чтения
```

### DocumentWorldObject (MonoBehaviour + IInteractable)

Вешается на GameObject в сцене вместо `WorldItemAdapter`.

```
DocumentWorldObject
├── data                DocumentData
├── PromptText          → data.interactPrompt
└── Interact()          → DocumentManager.Instance.Read(data)
                           + деактивирует подсказку (объект остаётся в сцене,
                             просто prompt пропадает при повторном наведении)
```

### DocumentManager (синглтон)

```
DocumentManager
├── foundDocuments      HashSet<string>   IDs найденных документов
├── Read(DocumentData)  показывает UI, добавляет в foundDocuments
├── IsFound(string id)  bool
└── GetArchive()        List<DocumentData> отсортированный список для меню
```

### DocumentReaderUI

Полноэкранный оверлей поверх HUD. Активируется через `DocumentManager`.

```
DocumentReaderUI
├── Show(DocumentData)
│   ├── пауза движения игрока (не Time.timeScale — только Input)
│   ├── fade in оверлей
│   ├── запустить TypewriterEffect для text
│   └── если Audio-тип → воспроизвести audioClip вместо текста
└── Hide()
    ├── fade out
    └── вернуть управление игроку
```

## Триггеры и поведение в сцене

### Обычная записка / журнал

**Объект в сцене:** меш бумаги / тетради с коллайдером на слое `Interactable`.  
**Компонент:** `DocumentWorldObject`.  
**Триггер:** игрок смотрит на объект → `InteractionManager` видит его → показывает prompt → игрок жмёт E.

После первого чтения:
- `PromptText` возвращает `null` или пустую строку → `InteractionPromptUI` ничего не показывает.
- Объект **не уничтожается**, остаётся декорацией в сцене.

### Аудиокассета

**Объект в сцене:** маленький физический объект, можно сделать Rigidbody.  
**Триггер:** тот же (`InteractionManager`), prompt — «Послушать».  
**При взаимодействии:** аудио запускается через `AudioSource` на `DocumentManager`, игрок сохраняет управление. В углу экрана — маленький индикатор «▶ запись воспроизводится».  
**Объект** после взаимодействия деактивируется (кассету «вставили»).

### Составной документ (дневник Горина)

Три отдельных `DocumentWorldObject` в разных точках сцены, у всех `groupId = "gorin_diary"`, `totalParts = 3`.

В архиве:
- Пока найдена одна часть — показывается «Дневник Горина [1/3]», читать можно только её.
- Когда найдены все — запись объединяется в одну с заголовком «Дневник Горина [полный]».

Визуально в мире: все три объекта выглядят одинаково (та же тетрадь), но лежат в разных местах. Первая — на виду, вторая — спрятана, третья — в вентиляции.

### Документы за дверью (доступны позже)

Некоторые документы физически недоступны без прогресса (медблок Вороновой — дверь заблокирована шкафом, нужна механика сдвига). Никакого флага «заблокирован» в `DocumentData` не нужно — просто физический доступ к объекту в сцене.

## UI — DocumentReaderUI

### Визуал (PS1-стиль)

```
┌─────────────────────────────────────────────────────────┐
│  ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  │
│  ░                                                   ░  │
│  ░   МЕДИЦИНСКИЙ ЖУРНАЛ                              ░  │
│  ░   Н. Воронова                                     ░  │
│  ░   ─────────────────────────────────               ░  │
│  ░   15 ноября, 08:00                                ░  │
│  ░                                                   ░  │
│  ░   Горин и ещё четверо ввели X-77 этой ночью...   ░  │
│  ░                                                   ░  │
│  ░                                                   ░  │
│  ░                              [E] — закрыть        ░  │
│  ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  │
└─────────────────────────────────────────────────────────┘
```

- Фон — тёмный полупрозрачный оверлей, края чуть виньетированы.
- Шрифт — моноширинный, советский стиль (Courier / PT Mono).
- Заголовок отделён горизонтальной чертой из `─`.
- Эффект печатной машинки при первом чтении (~30 символов/сек). При повторном — текст появляется сразу.
- Если текст длиннее экрана — прокрутка колесом или стрелками, без пагинации.

### Архив документов (в меню паузы)

Вкладка «Документы». Список слева, текст справа.  
Найденные — белые. Не найденные — не показываются (не «???», просто отсутствуют).  
Составные документы — одна запись, которая разворачивается по найденным частям.

## Нарративные триггеры

Некоторые документы влияют на состояние мира при первом прочтении:

| Документ | Триггер при чтении |
|---|---|
| Журнал Свиридова, запись №4 | Разблокируется запись в архиве Яруса В |
| Дневник Горина, часть 3 | Ambient-звук в лаборатории меняется (кристалл «реагирует») |
| Записка на двери реакторного отсека | Подсвечивается маршрут к генераторам (опционально) |
| Протокол Субъекта 0 | Разблокируется карантинный блок В-1 (Зимин роняет ключ — скриптовая сцена) |

Реализация: `DocumentManager.OnDocumentRead` — C# event, на который подписываются нужные менеджеры сцены.
